@apps-in-toss/web-framework 실제 API — node_modules .d.ts에서 직접 검증 (2026-04-13)
이 파일과 다른 문서가 충돌하면 이 파일이 우선한다.
존재하지 않는 API는 절대 생성하지 말 것 ("모르면 지어내지 마라").

=== 존재하지 않는 것 (생성 금지) ===
- useTossLogin, useTossPayment, useTossAd, useTossPromotion — React 훅 자체가 SDK에 없음
- window.Toss.xxx — 구식/환각 패턴
- TossRewardAd, AdSlot — SDK 컴포넌트 아님 (src/components/에 로컬 래퍼로 구현)

=== import 경로 ===
대부분: import { ... } from '@apps-in-toss/web-framework'
광고 저수준 API: import { TossAds, loadFullScreenAd, showFullScreenAd } from '@apps-in-toss/web-bridge'
(web-framework가 re-export하지만 TypeScript moduleResolution:"bundler"에서 타입 체인이 끊길 수 있음 → web-bridge 직접 import)

=== 광고 — 리워드/전면 (loadFullScreenAd + showFullScreenAd) ===
import { loadFullScreenAd, showFullScreenAd } from '@apps-in-toss/web-bridge'

loadFullScreenAd({ options: { adGroupId: string }, onEvent: (e: { type: 'loaded' }) => void, onError: (err: unknown) => void }): () => void (cleanup)

showFullScreenAd({ options: { adGroupId: string }, onEvent: (e: ShowFullScreenAdEvent) => void, onError: (err: unknown) => void }): () => void (cleanup)

ShowFullScreenAdEvent 타입:
  { type: 'requested' }
  { type: 'show' }
  { type: 'impression' }
  { type: 'clicked' }
  { type: 'dismissed' }          ← 광고 닫힘
  { type: 'failedToShow' }
  { type: 'userEarnedReward', data: { unitType: string, unitAmount: number } }  ← 리워드 지급 시점

사용 패턴:
  1. loadFullScreenAd → onEvent('loaded') 후
  2. showFullScreenAd → userEarnedReward 이벤트 수신 → dismissed 이벤트로 종료 확인
  반환된 cleanup 함수를 useEffect return / 컴포넌트 unmount 시 반드시 호출
